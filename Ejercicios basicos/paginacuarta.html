VENTAJAS Y DESVENTAJAS DE (XHTML,HTML5)

Ventajas de HTML5.

1.Es nativo, y por tanto independiente de plugins de terceros. Es decir, no pertenece a nadie, es opensource.

2.Es más semántico, con etiquetas que permiten clasificar y ordenar en distintos niveles y estructuras el contenido. Además, incorpora metadatos de manera más formal, favoreciendo el posicionamiento SEO y la accesibilidad.

3.El código es más simple lo que permite hacer páginas más ligeras que se cargan más rápidamente favoreciendo la usabilidad y la indexación en buscadores.

4. Ofrece una compatibilidad mayor con los navegadores de dispositivos móviles.

5. Incluye la etiqueta de dibujo canvas, que ofrece más efectos visuales.

6.Ofrece soporte a codecs específicos.

7.Posibilita la inserción de vídeos y audio de forma directa.

Desventajas de HTML5.

1.A veces, lleva tiempo tener listas las aplicaciones móviles debido a que los desarrolladores tienen que escribir código para cada plataforma.

2.Los desarrolladores trabajan en lenguajes que no son nativos para las plataformas; por eso, la eficacia del código depende del motor de traducción. Sucede a menudo que las técnicas de codificación son ineficientes o que el código es redundante.

3. La diferencia de tiempos de ejecución de las plataformas a menudo causa complicaciones en el desarrollo de aplicaciones móviles multiplataforma.

4.El estándar HTML5 evolucionó en un período relativamente corto y ello produjo discrepancias en la implementación de atributos CSS, etiquetas HTML y API de JavaScript. A veces, estas características se comportan de manera diferente en plataformas distintas. Sin embargo, hay herramientas disponibles para resolver estas discrepancias.

Ventajas de XHTML.

1.Compatibilidad parcial con navegadores antiguos: la información se visualiza, aunque sin formato.

2.Cabe apuntar que el XHTML 1.0 fue diseñado expresamente para ser mostrado en navegadores que soportan HTML de base.

3.Un mismo documento puede adoptar diseños radicalmente distintos en diferentes dispositivos, pudiendo incluso escogerse entre varios diseños para un mismo medio.

4.Facilidad de edición directa del código y de mantenimiento.

5.Formato abierto, compatible con los nuevos estándares que actualmente está desarrollando el W3C como recomendación para futuros agentes de usuario o navegadores.

6.Los documentos escritos conforme a XHTML 1.0 pueden potencialmente presentar mejor rendimiento en las actuales herramientas web que aquellos escritos conforme a HTML.

Desventajas de XHTML.

1.Hay pocos editores de ayuda que escriban correctamente el código xhtml.

2.La rigidez también puede ser una desventaja, ya que si hay algún error de sintaxis, no se mostrará en el navegador, pero si está hecho en html , se mostrará un error en el navegador.
